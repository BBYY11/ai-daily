📖 怎么给 AI Agent / LLM 用(ChatGPT / Claude / Gemini)

为什么 RSS 不够用?

RSS 协议(1999 年的)对人类阅读器很友好,但很多 LLM 的 web 工具解析 XML 不稳——ChatGPT 尤其。所以我们提供了 3 种格式让你挑:

  • RSS XML — 给 RSS 阅读器
  • JSON Feed — 给 LLM / Agent / 任何程序(强烈推荐)
  • Markdown — 直接粘到任何聊天 / 笔记 / 文档

给 ChatGPT / Claude 的具体指令模板

把下面这段粘到任何 LLM 聊天框(改 URL 为最新的):

请 fetch https://bbyy11.github.io/ai-daily/feed.json 这是 AI Daily 每天的早报 JSON Feed。请: 1. 总结今天(2026-06-04)AI 圈 5 大要闻 2. 重点分析"新兴"分类的项目(GitHub 涨星快 / HN 红的) 3. 找出本周 1 个被低估的趋势 + 给我 3 句话的论据 4. 用中文输出,简洁有力

或者用更结构化的提示词:

访问 https://bbyy11.github.io/ai-daily/feed.json 我要做一份"过去 7 天 AI 圈"简报,给我按: - 头部动态(Anthropic / OpenAI / NVIDIA / DeepSeek 等) - 新兴势力(GitHub / HN / Product Hunt 冒出的项目) - 监管 / 政策变化 - 中国 / 美国 双线对比 每块 3-5 条,要带数据(估值 / 涨星 / HN 点数 等)。

在 n8n / Make / LangChain 里订阅

任何一个支持 RSS 触发器的自动化工具,直接输入 feed.xml URL 即可。HTTP GET 频率建议 1 小时一次(你设的 <ttl>60</ttl> 提示 60 分钟)。

为什么 ChatGPT 之前说"读不到内容"?

三种可能 + 我的修复:

  1. ChatGPT 当时 fetch 失败(网络 / 限流)— 换 JSON Feed 试试,LLM 解析更稳
  2. ChatGPT 没有"订阅"功能(这是产品局限)— 订阅需要你每天手动去喂链接,或者用 n8n/IFTTT 这种中间件
  3. XML 被 LLM 当字符串读(常见)— 现在换成 JSON Feed,LLM 解析更精准
💡 真相:ChatGPT / Claude 本身都不提供"长期订阅 RSS"功能 — 这是产品形态决定的(它们是对话式 AI,不是信息流阅读器)。要用 LLM 长期监控,最稳的方法是:每天 8:30 把 feed.json 链接粘给它,问"今天有什么新动态";或者用 n8n 这种自动化工具订阅 + 触发 LLM 摘要。

3 种格式字段说明

每条新闻都包含这些字段(让 Agent 能做结构化分析):

  • id / title / summary / source / time / url — 基础元数据
  • category — "headline" / "rising" / "company" / "paper" / "industry" / "social"
  • category_label — 中文分类(头条/新兴/公司/学术/行业/声音)
  • tags — 自由文本标签数组
  • terms — 词条数组(可对照 terms.json 查百科)
  • heat — {score, level, sources[], breakdown} 多源热度
  • comments — 评论数估算
